# Windows installer (NSIS)

Thaw's Windows release is packaged with NSIS. This directory holds the **one
tracked, customised file** that shapes that installer; everything else here is
regenerated by `wails build --nsis` and is git-ignored.

| File | Tracked? | Purpose |
|------|----------|---------|
| `project.nsi` | ✅ | The installer script. Wails uses this instead of its stock template once present, and preserves it across builds. |
| `resources/header.bmp` | ✅ | Branded 150×57 24-bit BMP (Thaw glyph on the app's dark background) shown in the progress-page header. |
| `wails_tools.nsh` | ❌ generated | Macros + `INFO_*` defines, re-resolved from `wails.json` on every build. Do not edit or commit. |
| `tmp/MicrosoftEdgeWebview2Setup.exe` | ❌ generated | WebView2 bootstrapper embedded by Wails. |
| `../icon.ico` | ❌ generated | App icon, generated from `build/appicon.png`. |

## What the installer does (issue #825)

Instead of the classic gray, multi-page, Program-Files, UAC-prompting NSIS
wizard, `project.nsi` is a modern **one-click** installer:

- **No wizard pages** — a single branded, dark-themed progress screen.
- **Per-user, no UAC** — `RequestExecutionLevel user`; installs to
  `%LOCALAPPDATA%\Programs\Thaw` (the Electron convention). The uninstall entry
  is registered under `HKCU` (Add/Remove Programs still lists it).
- **WebView2 bootstrap** — silently installs the runtime if it is missing.
- **Auto-launch** — Thaw starts as soon as the copy finishes and the window
  closes itself.

The output artifact is still `thaw-<arch>-installer.exe`.

## Signing flow in CI (issue #820)

The release workflows (`build.yml` and `manual-release.yml`) sign in the
correct order so the binary embedded in the installer is signed too:

1. `wails build --nsis` (no `-obfuscated` on Windows — garble obfuscation
   trips Defender heuristics that hurt SmartScreen reputation) compiles
   `thaw.exe`, generates `wails_tools.nsh`, and produces a throwaway installer.
2. **Azure Trusted Signing** signs `build/bin/thaw.exe`.
3. `makensis` re-runs over `project.nsi` (passing only
   `-DARG_WAILS_<ARCH>_BINARY=…/thaw.exe`) to repackage the installer around
   the now-signed binary, overwriting the throwaway.
4. **Azure Trusted Signing** signs the installer itself.

Previously a single sign step ran after packaging, leaving the packed
`thaw.exe` unsigned.

## Editing / debugging locally

`project.nsi` is a plain NSIS script. To iterate without a full Wails build,
first run one `wails build --target windows/amd64 --nsis` to populate
`wails_tools.nsh`, then from this directory:

```bat
makensis -DARG_WAILS_AMD64_BINARY=..\..\bin\thaw.exe project.nsi
```

## Regenerating the header bitmap

`resources/header.bmp` must be an **uncompressed 24-bit BMP3** (the format the
NSIS MUI renderer expects — 32-bit `BITMAPV5` files render with wrong colours).
It is composed from `build/appicon.png` (Thaw glyph, right-aligned, on the
`#0d1117` brand background). Regenerate it only when the app artwork changes.
